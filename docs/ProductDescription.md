# 一、产品概述

> 产品名称：**Schema**

## 1.1 产品背景

随着个人知识获取渠道的碎片化与信息密度的持续上升，单纯的笔记工具已难以满足用户对“长期知识沉淀、结构化管理与持续复用”的需求。现有以 Confluence 为代表的知识管理平台在信息模型与治理能力上较为成熟，但其设计目标主要面向团队协作与组织管理，在个人使用场景下存在功能冗余、复杂度过高等问题。

因此，有必要设计一款**面向个人的知识管理系统**，在理念与使用方式上与 Confluence 保持一致，但彻底去除协作与组织属性，聚焦个人长期知识资产的构建与演进。

## 1.2 产品定位

本产品定位为：**面向个人的结构化知识管理与思考沉淀工具**。

其核心价值不在于“记录”，而在于帮助用户建立清晰、可扩展、可回顾的个人知识体系，使知识能够随着时间不断演进，而不是被动堆积。

## 1.3 目标用户

产品的目标用户包括但不限于：

* 技术从业者，用于技术学习、方案沉淀与经验总结
* 研究人员与学生，用于知识整理、文献记录与长期研究
* 知识密集型工作者，用于项目复盘、方法论沉淀与个人成长记录

# 二、设计原则

在整体设计上，产品遵循以下原则：

* 单人优先：所有功能围绕“单一使用者”的长期体验设计
* 结构先行：通过稳定的信息模型降低知识规模增长后的混乱风险
* 低摩擦创作：减少记录与编辑的心理与操作成本
* 长期可用：强调检索、回顾与再利用，而非短期记录效率

# 三、信息模型与核心概念

## 3.1 页面（Page）

页面是系统中最小的知识单元，用于承载具体内容，包括但不限于：笔记、方案、想法、总结、清单等。页面支持富文本与结构化内容混合编辑，并作为知识关联与引用的基本节点。

页面可以直接编辑，并实时渲染，确保用户在阅读与创作之间无缝切换。

## 3.2 知识库（Library）

知识库是页面的逻辑集合，用于对个人知识进行主题域划分，例如“技术学习”“工作项目”“个人成长”等。每个知识库内部支持树状层级结构，用于表达知识的上下位关系。

知识库从本质上来说也是一个页面，用户可以为其添加描述、标签与元数据等信息，以增强其表达能力。与页面不同的是知识库不支持被设置为其他页面的子页面。

## 3.3 标签与元数据

标签用于对页面进行横向分类，支持跨知识库使用。元数据包括创建时间、更新时间以及可扩展的自定义字段，用于后续筛选、检索与视图构建。

## 3.4 页面关联

系统支持页面之间的引用与反向引用，形成知识网络结构，帮助用户在不同知识之间建立明确联系，避免孤立信息点的长期存在。

# 四、核心功能设计

> 本章节所描述功能均以 Schema（个人知识管理系统）为产品主体。

## 4.1 内容编辑与表达

系统提供高性能所见即所得编辑器，支持常见富文本能力，以 Markdown 语法为基础扩展丰富的内容表达形式。

编辑与阅读状态可自然切换，确保在长时间阅读与频繁编辑场景下依然保持良好体验。

## 4.2 组件与宏机制

引入可复用的内容组件机制，用于增强页面的结构化表达能力。典型组件包括：

* 状态标识组件，用于标注页面或内容块的阶段状态
* 任务组件，用于记录后续行动点
* 引用组件，用于关联其他页面或外部资料
* 目录组件，用于快速定位页面结构

组件设计强调轻量与稳定，避免过度复杂化。

## 4.3 模板系统

系统内置并支持自定义模板，用于固化常见知识结构，如：

* 读书笔记模板
* 技术调研模板
* 项目复盘模板
* 周期总结模板

模板的目标是降低创作门槛并提升知识结构一致性。

## 4.4 知识组织与演进

系统同时支持树状层级与跨页面引用，确保用户既可以自上而下构建体系，也可以自下而上积累内容。

页面历史版本自动保存，用户可随时回溯过往内容，用于支持知识的长期演进与修订。

## 4.5 搜索与发现

提供全文搜索能力，覆盖标题、正文、标签与组件内容，并支持按时间、知识库、标签进行过滤。

在此基础上，系统提供多种结构化视图，如最近更新、按标签聚合、长期未访问页面等，辅助用户从不同角度理解自身知识体系。

## 4.6 回顾与复用机制

系统支持轻量回顾功能，例如：

* 最近修改内容提醒
* 历史同日内容回顾
* 周期性知识回顾视图

其目标在于提升既有知识的再利用率，而非单纯鼓励新内容输入。

## 4.7 任务与行动支持

页面内可创建轻量任务，用于标记需要后续行动的内容。所有任务可汇总至个人待办视图，但不提供复杂的流程、状态流转或管理能力。

## 4.8 知识库与页面公开访问

支持将**指定知识库或单个页面设置为公开访问状态**，以满足个人知识对外展示、分享与引用的需求。

公开访问具备以下特征：

* 公开内容无需登录即可访问
* 访问权限默认为只读，不支持外部编辑或评论
* 可独立控制知识库级与页面级的公开状态
* 私有内容与公开内容在逻辑与存储层面保持隔离

该能力的目标是支持“个人对外知识输出”，而非协作或共建。

# 五、非功能性设计

## 5.1 权限与安全

系统为单用户模式，不提供复杂权限体系。重点放在数据安全、隐私保护与用户可控性上。

## 5.2 数据备份与导出

支持数据自动备份与手动导出，导出格式包括 Markdown、PDF 等常见形式，以增强用户对长期使用的信心。

## 5.3 扩展与集成

系统预留扩展能力，用于未来集成浏览器剪藏、外部资料引用或其他个人工具、页面组件等，但不以企业级系统集成为设计重点。

# 六、产品边界说明

本产品不提供以下能力：

* 多人协作与实时协同编辑
* 成员、组织与权限管理
* 审批流、流程引擎与工作流管理

所有功能设计均围绕“个人长期知识管理”这一单一目标展开。

# 七、总结

该产品本质上是对 Confluence 思想的个人化重构：保留其成熟的信息模型与知识治理理念，去除团队协作复杂度，强化长期思考、结构化沉淀与知识演进能力，最终形成一款真正服务于个人认知成长的知识管理系统。
